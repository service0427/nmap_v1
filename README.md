# Naver Map Auto-Simulation Single-Mode (V1.0.0)

본 인프라는 안드로이드 단말기를 활용하여 네이버 지도(Naver Maps)의 가상 주행 및 트래픽 시뮬레이션을 수행하는 **완전 무인 싱글모드 오케스트레이션 시스템**입니다.

## 🚀 V1.0.0 핵심 아키텍처 (Single-Mode)

기존의 복잡한 인프라 의존성을 제거하고, 각 단말기가 독립적으로 최적의 시뮬레이션을 수행할 수 있도록 설계되었습니다.

### 1. Pure Dynamic UI (NO Hardcoding)
- 모든 UI 조작은 실시간 XML 덤프 분석을 통해 이루어집니다. 하드코딩된 좌표를 사용하지 않으며, 텍스트 및 리소스 ID 매칭을 통해 다양한 기기 해상도에 완벽히 대응합니다.
- **Visual-Structural Audit**: 모든 클릭 시점의 화면을 `.png`와 멀티라인 `.xml` 쌍으로 기록하여 완벽한 사후 분석을 보장합니다.

### 2. Intelligent Task API (V3.0 Protocol)
- **실시간 작업 할당**: 중앙 API 서버로부터 목적지 정보 및 고유 식별 정보(Identity)를 할당받아 즉시 시뮬레이션에 주입합니다.
- **정밀 통계 수집**: 실제 주행 패킷을 실시간 분석하여 주행 거리(m)와 시간(s)을 DB에 정확히 기록합니다.

### 3. Identity Washing & Randomized IP
- **Washing**: SSAID, ADID, NI, IDFV, TOKEN 등 모든 개인 식별 정보를 실시간 위조 데이터로 치환합니다.
- **Randomized Rotation**: LTE 동글의 IP를 120~180분 사이의 랜덤 주기로 자동 회전시켜 탐지를 원천 봉쇄합니다.

### 4. Self-Healing & Safety Monitor
- **네트워크 자가 복구**: 연결 단절 시 라우팅 테이블 재구성 및 USB 물리 리셋을 자동으로 수행합니다.
- **다중 안전 장치**: 15분 절대 타임아웃, 하트비트 감시, 고스트 앱 탐지 로직을 통해 좀비 프로세스 발생을 차단합니다.

## 📂 프로젝트 구조
- `wifi_single/`: V1.0.0 싱글모드 핵심 엔진 및 오케스트레이터.
- `utils/`: IP 회전, 웹 모니터링, 라우팅 복구 등 공용 유틸리티.
- `device_init/`: 단말기 환경 자동화 구축 및 Magisk 모듈 설치 도구.
- `cmd/`: 단말기 일괄 제어(밝기, 볼륨, 재부팅 등) 스크립트.
- `scrcpy/`: 원격 화면 제어 및 정렬 유틸리티.

---

## 🛠 `device_init.sh` : 단말기 초기 구축 및 최적화
ADB로 연결된 모든 기기에 대해 아래 과정을 일괄 수행합니다.
1. **시스템 최적화**: 블루투스 비활성화, 사운드 음소거, 재난문자 차단 등 시뮬레이션 방해 요소 제거.
2. **Magisk & Zygisk 구축**: Frida 주입 및 시스템 인증서 설치를 위한 루트 권한 환경 자동 구성.
3. **인증서 복구**: 사용자 인증서 해시 비교를 통해 필요 시에만 재설치 및 재부팅을 수행하는 최적화 로직 적용.

---

## 🔧 `cmd.sh` : 일괄 원격 제어 유틸리티
- **`--light`**: 모든 기기의 화면 밝기, 볼륨, 애니메이션 배율을 시뮬레이션 최적값으로 일괄 복구합니다.
- **`--reboot`**: 연결된 모든 기기를 안전하게 재부팅합니다.
- **`--wifi`**: 주변 Moon 및 U26- 와이파이를 탐색하여 최적의 SSID로 일괄 연결하고 확인 팝업을 자동 해결합니다.

> **주의사항**
> 모든 주행 데이터와 덤프 파일은 기기별 로그 폴더(`wifi_single/logs/{DEV_ID}/`) 내부에 엄격히 격리되어 저장됩니다.
